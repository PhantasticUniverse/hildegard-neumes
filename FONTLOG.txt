FONTLOG for Hildegard Neumes

This file provides detailed information on the Hildegard Neumes font.
This information should be distributed along with the Hildegard Neumes
font and any derivative works.

Basic Font Information
----------------------

Hildegard Neumes is an OpenType font for the Rhineland neume notation
used in Hildegard von Bingen's music manuscripts — the Dendermonde Codex
(c. 1174/75) and the Wiesbaden Riesencodex Hs. 2 (c. 1180–85). It is
designed as a drop-in replacement for the experimental inline-Rust
Rhineland glyph module in the Rhena/Viriditas notation platform, and
as a universally consumable OpenType artifact for any chant tool that
wants paleographically faithful 12th-century German neume shapes.

The font's v1 atomic inventory contains 19 glyphs covering puncta,
virgae, quilismae, oriscus, strophicus, pressus, liquescents, clefs,
division marks, and connector lines. Multi-component neume families
(pes, flexa, climacus, torculus, porrectus, and the Hildegard-idiosyncratic
compounds) are assembled by the consumer's resolver from these atoms.

Glyph shapes are traced from direct reads of the manuscript facsimiles
at Wiesbaden HLB RheinMain (Riesencodex, CC-BY 4.0) and the IMSLP
Dendermonde facsimile (public domain), following the paleographic
register documented in Gardiner 2022, van Poucke's Alamire facsimile
introduction, and direct folio inspection.

ChangeLog
---------

19 April 2026 (v0.1.0-dev) — Hildegard Neumes Project maintainers
- Phase A UFO3 scaffold landed. src/hildegard-neumes.ufo/ now exists
  with 20 glyph slots (.notdef + 19 Rhineland atoms), placeholder
  rectangle outlines at the post-review advance widths, SMuFL
  codepoints in the cmap, and public.glyphOrder pinned. Produced by
  scripts/scaffold-ufo.py from the 19-atom contract. Round-trip
  verified by tests/test_scaffold.py (14 cases: file structure,
  UFOReader round-trip, scaffolder guardrails).
- Initial git commit captured the pre-scaffold infrastructure state
  (59 files, eight ADRs, six CI workflows, 42 pytest cases).
- Doc sweep closed stale references: rhena_integration_plan.md width
  table updated to post-review values, svgpathtools mentions removed,
  widths_proposed.json pointers replaced with rhineland.contract.json,
  paleographic drawing brief section headers aligned to corrected
  widths, glyph priority sheet tier widths refreshed.

14 April 2026 (v0.1.0-dev) — Hildegard Neumes Project maintainers
- Initial project scaffolding.
- 19-glyph atomic inventory defined (rust_const / font_name / width /
  SMuFL cross-reference), matching Rhena's existing Glyph struct API.
- UFO3 source target established (ADR-0001). Headless FontForge build
  pipeline with SOURCE_DATE_EPOCH=0 for reproducibility.
- Python codegen pipeline to Rhena implemented (ADR-0002): parametrized
  test suite over both TTF (quadratic) and OTF/CFF (cubic) fixtures.
- Determinism strategy committed (ADR-0004): round-half-up integer
  coordinates, stripped head.created/modified sha256, double-build CI
  byte-identity gate.
- Width review found and corrected three overflow bugs in the inherited
  widths (rh_virga 65→90, rh_liquescent_asc / desc 140→160). Contract
  now carries the corrected values; Rhena catches up via a coordinated
  adoption PR.
- OFL 1.1 licence adopted with Reserved Font Name "Hildegard Neumes"
  (ADR-0006).
- No glyph outlines yet authored. Drawing phase unblocked on all
  infrastructure fronts; begins after UFO3 source scaffolding in
  FontForge and the coordinated Rhena-side width PR.
